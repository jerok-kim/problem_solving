# 문제풀이를 위한 입출력

## 입력

### 1. 한 줄 입력받기

가장 기본적인 입력방식이다. 입력이 문자열 한 줄만 주어질 때 쓰면 좋다.

```rust, noplayground
use std::io::stdin;

fn main() {
    let mut buffer = String::new();
    stdin().read_line(&mut buffer).unwrap();
    
    // ...
}
```

### 2. 여러 줄 순차적으로 입력받기

여러 줄에 걸쳐서 문자열이 입력으로 주어질 때 쓸 수 있는 방식이다. 매번 새 라인을 입력받을 때마다 버퍼를 클리어해줘야하기 때문에 효율적인 방식은 아니다. 대신 메모리가 많이 들지 않기 때문에 메모리가 중요한
경우에는 유용하다.

```rust, noplayground
use std::io::stdin;

fn main() {
    let mut buffer = String::new();
    stdin().read_line(&mut buffer).unwrap();
    
    let n = buffer.trim().parse::<usize>().unwrap();
    
    for _ in 0..n {
        buffer.clear();
        stdin().read_line(&mut buffer).unwrap();
        
        // ...
    }
}
```

### 3. 숫자 여러개 입력받기

정수 숫자를 띄어쓰기나 줄바꿈으로 구분해서 여러개 입력받아야 하는 경우에, 라인별로 입력, 스플릿, 파싱하는 것이 아니라 전체를 입력받아서 한꺼번에 스플릿, 파싱해서 저장해둔다. 그리고 필요할 때
마다 `input.next().unwrap()`으로 하나씩 뽑아서 쓴다.

입력을 한 번에 버퍼에 저장하기 때문에 입력이 아주 클 경우 메모리 이슈가 발생할 수 있다. 그리고 커맨드라인에서 직접 입력 데이터를 줄 수가 없고 파일에 써서 사용해야 한다.

인풋 데이터를 스플릿할때 `split_whitespace()`대신 `split_ascii_whitespace()`를 사용한다. 후자가 좀 더 빠르다고 하는데, 확인해보자. 문제에서 대부분 공백 혹은 `\n`만으로
데이터가 구분되기 때문에 `split_ascii_whitespace()`를 쓰는 것이 좋다.

```rust, noplayground
use std::io::{stdin, Read};

fn main() {
    let mut input = String::new();
    stdin().read_to_string(&mut input).unwrap();
    let mut input = input.split_ascii_whitespace().flat_map(str::parse::<usize>);
    
    let n = input.next().unwrap();
    
    // ... 필요할 때 마다 input.next().unwrap() 호출
}
```

### 4. 문자열 여러개 입력받기

3번에서 파싱하는 부분을 제외한 버전이다. 공백문자로 구분된 데이터들이 숫자가 아니라 문자열이 포함된 경우 사용한다.

```rust, noplayground
use std::io::{stdin, Read};

fn main() {
    let mut input = String::new();
    stdin().read_to_string(&mut input).unwrap();
    let mut input = input.split_ascii_whitespace();
    
    let n = input.next().unwrap().parse::<usize>().unwrap();
    let line = input.next().unwrap();
    
    // ... 필요할 때 마다 input.next().unwrap() 호출
}
```

### 5. 입력 데이터 파일로 받기(표준 입력 리디렉션)

커맨드라인에서 실행시킬 때 별도의 파일에 인풋 데이터를 적은 다음에 다음과 같은 커맨드를 실행시키면 파일에서 입력을 받을 수 있다.

```text
$ cargo run < input.txt
```

부등호 방향을 거꾸로 하면 파일에 출력도 가능하다. Clion IDE에서 실행 설정에서 입력 리디렉션을 설정하면 된다.

## 출력

### 1. 한 줄 출력하기

내부적으로 `stdout`에 매번 락을 걸고 해제하기 때문에 여러 줄을 반복적으로 출력하는 경우에 매우 비효율적이다.

```rust, noplayground
let data = 45;
println!("{data}");
```

### 2. BufWriter를 이용해 여러 줄 출력하기

출력해야 하는 양이 많을 때 사용하면 좋다. 버퍼를 사용하기 때문에 출력속도가 좋아지는 것 외에도 `BufWriter`가 생성될때 락을 걸고, 생명주기가 끝날때 까지 유지되기 때문에 `println!()`매크로를
사용해 출력할때처럼 매번 락을 걸었다 풀었다 하지 않는다.

```rust, noplayground
use std::io::{stdout, Write, BufWriter};

fn main() {
    let stdout = stdout();
    let mut writer = BufWriter::new(stdout);
    
    for i in 0..100_000 {
        writeln!(writer, "{i}").unwrap();
    }
}
```

### 3. String 인스턴스를 버퍼로 활용하기

출력이 한 줄 이상일때 유용하다. 2번 방식과 비슷하지만, `String`인스턴스를 버퍼로 활용한다는 것이 차이점이다. `BufWriter`처럼 버퍼가 다 차면 쓰는 것이 아니라 출력해야할 것들을
일단 `String`에 다 쓰고 한 번에 `stdout`으로 출력하기 때문에 메모리가 얼마나 필요할지 예상할 수 없다는 단점이 있다. 대부분의 경우 메모리가 큰 문제가 되지 않으므로 2번 방식보다 좀 더 빠른 3번
방식이 좋을 수 있다.

```rust,noplayground
use std::fmt::Write;

fn main() {
    let mut output = String::new();

    for i in 0..100_000 {
        writeln!(output, "{i}").unwrap();
    }

    println!("{output}");
}
```

러스트는 C/C++ 수준으로 빠릅니다. 빡빡한 시간 제한이 있는 알고리즘 문제를 푼다면 러스트는 괜찮은 선택지입니다. 그런데 러스트로 문제를 풀다보면 좀 성가신 부분도 있습니다. 입력 파싱은 쉽게 너저분해지고,
조심하지 않으면 실행 시간의 대부분을 출력에 쓰게 됩니다. 다른 언어에서 마음껏 사용하던 전역 변수를 못 쓰니 그 변수들을 모두 함수의 인자로 넘겨주기 시작하고, 함수 인자가 자꾸 늘어납니다. 이 글은 러스트로
알고리즘 문제를 풀 때 흔히 겪는 문제와 해결책을 소개합니다.

입출력
한 번에 입력 받기
입력은 두 가지 이유로 가능하면 한 번에 String에 담는 것이 좋습니다. 먼저 Read 트레잇의 메서드보다 String의 메서드가 더 다양하고 쓰기 편합니다. 두 번째로 시스템 호출이 줄어들어 입출력 시간을
절약할 수 있습니다. 입력을 한 번에 String에 담으려면 아래처럼 하면 됩니다.

```rust
use std::io::{self, Read};

fn main() {
    let mut buf = String::new();
    io::stdin().read_to_string(&mut buf).unwrap();
// 여기서 `buf`를 사용하면 됩니다. `buf`에는 입력 전체가 담겨있습니다.
}
```

메모리 제한이 있는 일부 문제에서는 한번에 입력을 다 받아오기 어려울 수 있는데, 이런 경우에는 BufReader를 이용하여 적절한 크기의 버퍼를 설정하면 됩니다. BufReader를 사용하지 않더라도 러스트 표준
라이브러리는 8 KB의 버퍼를 가지고 입력을 처리합니다. 즉, 버퍼의 크기를 바꿔가며 실행 시간을 줄여보려고 한다면 적어도 8 KB보다는 크게 설정해야 합니다.

깔끔하게 입력 파싱하기
러스트는 C의 scanf() 같은 함수가 없는데다 명시적인 예외 처리를 요구하기 때문에 입력 파싱을 위해 아래처럼 같은 코드를 반복해서 작성하게 됩니다.

```rust
let mut slices = buf.split_whitespace();

let i = slices.next().unwrap().parse::<i32>().unwrap();
let j = slices.next().unwrap().parse::<i32>().unwrap();
let f = slices.next().unwrap().parse::<f32>().unwrap();
let d = slices.next().unwrap().parse::<f64>().unwrap();
```

이럴 때는 아래처럼 매크로를 선언하고, 호출하는 시점에서 타입 추론을 유도하면 반복되는 코드를 줄일 수 있습니다.

```rust
let mut slices = buf.split_whitespace();

macro_rules! next {
() => { slices.next().unwrap().parse().unwrap() };
}

let i: i32 = next!();
let j: i32 = next!();
let f: f32 = next!();
let d: f64 = next!();
```

한 번에 출력하기
여러 줄에 걸친 출력을 수행할 때에는 println!()의 사용을 최소화해야 합니다. println!()을 사용하면 출력이 버퍼링되지 않기 때문입니다. 러스트 표준 라이브러리는 표준 출력을 줄 단위로 버퍼링하기
때문에 (새줄 문자를 출력하는) println!()이 사용될 때마다 버퍼의 내용을 모두 내보냅니다. 너무 빈번한 내보내기는 긴 실행 시간으로 이어질 수 있습니다.

간단한 해결책은 출력을 String에 모았다가 한번에 내보내는 것입니다. writeln!() 매크로를 사용하면 println!() 매크로처럼 표준 포맷팅을 사용할 수 있어 간편합니다.

```rust
use std::fmt::Write;

fn main() {
    let mut buf = String::new();

    writeln!(buf, "{}", 42).unwrap(); // buf는 "42"가 됩니다.

    println!("{}", buf); // 실제 출력은 한번에 수행합니다.
}

```

입력에서와 마찬가지로, 메모리 초과 문제가 발생하는 경우에는 한 번에 출력하는 대신 BufWriter를 사용하면 됩니다.

전역변수
러스트에서는 가변 전역변수의 사용이 상당히 까다롭습니다. 그래서 여러 함수가 상태를 공유해야하는 상황이 오면 (다른 언어에서는 으레 전역변수로 선언했을) 변수들을 인자로 넘겨가며 사용하다가 아래처럼 인자가 비슷비슷한
함수가 잔뜩 생기고 맙니다.

```rust
fn n_queen(ld: u16, col: u16, rd: u16, q: usize, n: usize) -> i32 { /* ommited */ }

fn n_queen_helper(ld: u16, col: u16, rd: u16, q: usize, n: usize) -> i32 { /* ommited */ }

fn n_queen_another_helper(ld: u16, col: u16, rd: u16, q: usize, n: usize) -> bool { /* ommited */ }
```

이런 경우에는 해당 변수들을 묶어 구조체를 정의합니다. 그러면 그 구조체의 메서드에서는 해당 변수들을 전역변수처럼 쓸 수 있습니다. (물론 self는 써야하지만요)

```rust
struct NQueen {
    ld: u16,
    col: u16,
    rd: u16,
    q: usize,
    n: usize,
}

impl NQueen {
    fn solve(&mut self) -> u32 { /* ommited */ }
    fn helper(&mut self) -> u32 { /* ommited */ }
    fn another_helper(&mut self) -> u32 { /* ommited */ }
}
```

또다른 입출력 방식 보기

```rust
use std::io;

fn main() {
    let mut s = String::new();

    io::stdin().read_line(&mut s).unwrap();

    let values: Vec<i32> = s
        .as_mut_str()
        .split_whitespace()
        .map(|s| s.parse().unwrap())
        .collect();

    println!("{}", values[0] + values[1]);
}
```

```rust
use std::io::Read;

fn read_more_fast() -> String {
    let mut buf = String::new();
    loop {
        let input = std::io::stdin().bytes().next().unwrap().unwrap() as char;
        
        if input == '\r' {
            continue;
        }
        
        if input == ' '||input=='\n' {
            return buf;
        }
        buf.push(input)
    }
}
```
